## Task 1: Introduce Intent-Sensitive Lethal Penalty in Environment
Standard MAPPO catches up because avoiding teammates is a static physical rule it can memorize. We need to introduce a dynamic rule tied to the Intent vector `I`.
Please update the `reward()` function in `uav_scheduling_env.py`:
1. **Define a Radar Threat Zone:** Assign a "Radar Threat Zone" (e.g., radius `R_threat = 0.3`) to a subset of targets (or fixed points in the map).
2. **Intent-Sensitive Logic:** 
   - Use the first dimension of the intent vector `I[0]` as the **Tactical Posture Flag** (e.g., `I[0] > 0.5` means "Aggressive/Attack", `I[0] < 0.5` means "Stealth/Evade").
   - If a UAV enters the `R_threat` zone while `I[0] < 0.5` (Stealth mode), apply a **massive penalty** (e.g., `reward -= 5.0`) to simulate being shot down. 
   - If a UAV enters the zone while `I[0] > 0.5` (Attack mode), apply a normal task progress reward.
   - For standard MAPPO (which runs with a fixed/dummy intent vector in our baseline setup), it will inevitably wander into the radar zone and get destroyed during Stealth phases, permanently preventing its reward curve from catching up to I-MAPPO.
